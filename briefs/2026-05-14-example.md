# Brief — 2026-05-14-focusblock-launch-visuals

The first worked example for this repo. Paired with the rendered output in `examples.md` and the actual HTML artifacts in `examples/`. Deliberately industry-neutral so the example illustrates the architecture without bleeding into any specific operator's domain.

This brief intentionally uses the same product (FocusBlock) as the sibling `your-content-worker` repo's example. The two starter repos demonstrate two workers operating against the same product launch — prose worker produces the launch post; design worker produces the launch visuals. Studying both shows the same dispatch shape in two different domains.

---

## 1. What to produce

Three self-contained HTML artifacts, one launch sequence:

1. **Social card** — 1080×1080, HTML. For LinkedIn / X / IG feed at launch announcement.
2. **OG card** — 1200×675, HTML. For Open Graph / Twitter Card metadata on the FocusBlock site.
3. **Preview page** — 1280×900, HTML. A self-contained preview block showing the launch announcement as it might appear in a press kit or a partner-facing handoff.

All three are part of one sequence. Cross-artifact consistency rules in Section 3.

## 2. Audience

Remote knowledge workers — engineers, designers, writers, analysts — who manage their own time and have tried at least two productivity systems that didn't stick. They're skeptical of habit-tracker apps and gamification. They scroll fast on LinkedIn and X; the social card will be seen at thumbnail size before anything else. They click through to the OG card when a friend shares the link; the OG card sets expectations for the landing page. They see the preview page if they're on the press-kit list.

## 3. Format and constraints

**Copy (verbatim across all three artifacts):**

- **Kicker:** `LAUNCH · v1.0`
- **Headline (the line):** `Schedule the focus. Stop renting the willpower.`
- **Sub:** `A focus-timer that learns your energy patterns and books focus on your calendar.`
- **CTA / URL:** `focusblock.example/download`
- **Footer meta (preview page only):** `Available · macOS · Windows · Linux · Free for solo use`

**Assets:**
- No logo provided for this dispatch. Use a typographic mark — the word `FocusBlock` set in display weight at 20-24px, no logo glyph. Compose as if logo refinement is a separate downstream dispatch.
- No product screenshots. Composition is pure type + grid + palette swatches.

**Accessibility floor:**
- Body-on-background contrast ≥ 4.5:1 (WCAG AA for body text).
- Headline-on-background contrast ≥ 3:1 (WCAG AA for large text).
- All visible text is real HTML text, not rasterized into images.

**Structural — Social card (1080×1080):**
- Kicker top-left at outer-padding inset.
- Headline left-aligned in the upper-middle band, anchored to col-1 of the 12-col grid.
- A row of three swatch chips (`--bg`, `--ink`, `--accent-warm`) sitting on the baseline grid in the lower-middle band, each labeled with hex code in mono.
- Sub line below swatch row.
- CTA bottom-left in mono.
- Faint vertical grid markers visible in the hero band only.

**Structural — OG card (1200×675):**
- Same kicker / headline / CTA arrangement but adapted to wider aspect.
- Headline dominates the left two-thirds of the frame.
- Right third holds a vertical specimen strip: three swatches stacked, each with hex code below.
- CTA bottom-left at outer-padding inset.

**Structural — Preview page (1280×900):**
- Top band: kicker, headline, sub.
- Mid band: a horizontal palette specimen — five swatches with hex codes and token names (`--bg`, `--ink`, `--muted`, `--accent-warm`, `--rule`).
- Lower band: a 3-pane feature row — three short paragraphs, each headed by a mono specimen label (`01 ── SCHEDULE`, `02 ── LEARN`, `03 ── PROTECT`).
- Footer: meta line, CTA on the right.

**Cross-artifact consistency:**
- Kicker, headline, sub, CTA copy verbatim identical.
- Palette tokens identical (use `reference/visual-system.md` defaults).
- Type families identical.
- Grid feel consistent: all three artifacts show specimen labels, all three reveal at least some grid markers, all three feel like they come from the same designer's tool tray.

## 5. Success criteria

- All three artifacts ship as self-contained HTML files at the exact specified pixel dimensions (set via `width` and `height` on the `<body>` or root container, plus a viewport meta and a `body { width; height; }` rule so the rasterizer captures the right frame).
- Every color used is from `reference/visual-system.md`'s palette table.
- Every font in use is in the typography table (system stack — no external font fetches).
- Body-on-background contrast ≥ 4.5:1, headline-on-background contrast ≥ 3:1, verified against the WCAG AA thresholds.
- All visible text is real HTML text, not images.
- `grep -E "https?://" examples/*.html` returns only the CTA URL (`focusblock.example/download`) and no external CSS / JS / font / image fetches.
- Files load via `file://` with zero network requests (verifiable in browser dev-tools network tab).
- Kicker, headline, sub, CTA copy match Section 3 verbatim across all three artifacts.
- Social card stays legible at thumbnail size (160px wide) — verified by zooming browser to 15%.
